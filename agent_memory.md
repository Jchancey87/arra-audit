## Critical Architectural Constraints (Red Lines)
- **YouTube Embedding**: `controls: 1` + `origin` in `playerVars`. Never `pointer-events: none` on iframe containers (blocks autoplay unlock gesture). **SUPERSEDED 2026-06-21**: YouTube IFrame has been removed. Audio is downloaded at import time and played via a single shared `<audio>` element.
- **Local audio architecture** (2026-06-21, **updated 2026-06-21 sync import**): All song playback goes through one `<audio ref={audioRef}>` rendered by `AudioContext`. wavesurfer in the arrangement timeline attaches via `WaveSurfer.create({ media: audioRef.current, ... })` so waveform + transport share one MediaElement (no drift, no second Web Audio). Songs have `sourceType: 'local'` after import; `publicUrl: '/uploads/songs/{songId}.{ext}'` is the browser-fetchable URL. **POST /api/songs/import is now synchronous**: it awaits the YouTube audio download and either returns 201 with `publicUrl` set, or purges the half-imported song + 502 on failure. There is no client-side polling and no `/songs/:id/audio-url` route. For legacy songs stuck with `publicUrl=null` (created before this fix), the client offers a "Re-download audio" button in `StudySessionWorkspace` that calls `POST /songs/:id/download-audio`. `audioStorageService` is the port for moving files into the uploads tree; `AudioDownloadService` owns the network call to Python's `/download` endpoint.
- **Study session playback surfaces (2026-06-21)**: `StudySessionWorkspace` hides the global `MonitorPortal` via `setShowVideo(false)` on mount and only renders the wavesurfer-backed `ArrangementTimelineWidget` for `arrangement`/`form` lens days — so on texture/harmony/rhythm/melody days there is no playback UI at all unless the Reference Signal panel's inline transport row is present. The Reference Signal panel carries an inline play/pause + ±10s + `M:SS / M:SS` row (added 2026-06-21) that calls the shared `useAudio()` transport — drives the same single `<audio>` element, no second engine, no drift. Buttons are disabled and the time read-out shows `no audio` when `activeSong?.publicUrl` is null. Arrangement/form days now have BOTH the inline row AND the wavesurfer timeline widget. **Do not remove the inline row** when refactoring; doing so will silently break playback on non-arrangement lens days.
- **Service Layering**: Business logic in `services/`, not routers. Swappable repos (`MongoSongRepository`, `InMemoryRepository`).
- **Service construction order in `server.js`**: `FilesystemAudioStorageAdapter` MUST be constructed *before* `SongService` and passed as its 4th arg. - **loadSong MUST be memoized (2026-06-21, the white-noise bug)**: `AudioContext.loadSong` was a plain `async` function (new identity every render). AuditForm had `useEffect(() => { if (song) loadSong(song); }, [song?._id, loadSong])` — because `loadSong` changed identity every render, the effect fired every render → `audioRef.current.pause()` + `currentTime = 0` on every render → the audio was yanked back to t=0 constantly. User symptom: "brief white noise then silence" (the first few ms of the MP3 before the next render cycle paused it). Fix: `useCallback([])` — safe because all referenced values are stable useState dispatchers + a stable ref. `setShowVideo` has the same class of bug (used as a useEffect dep in StudySessionWorkspace) and is now memoized too. **Never use a non-memoized AudioContext function as a useEffect dependency.**
- **Universal wavesurfer regions + timeline2 (2026-06-21)**: `UniversalWaveformBar.jsx` is the site-wide waveform component. It wraps `WaveformTimelineOverlay` (which owns `WaveSurfer.create` + `RegionsPlugin` + `TimelinePlugin`) + a compact transport row, and pulls everything from `useAudio()`. Rendered above the tab bar in `AuditForm` (all lens tabs + analysis) and in the Reference Signal panel of `StudySessionWorkspace` (all lens days). `WaveformTimelineOverlay` takes a generic `regions` prop: `{ id, start, end, color, label, drag, resize, selected }` — callers build regions from sections / bookmarks / tagged timestamps. `ArrangementTimelineWidget` has a `hideWaveform` prop — pass it when UniversalWaveformBar is also rendered above to avoid two `WaveSurfer.create()` instances on the same `<audio>`. Region colors: `LENS_REGION_COLOR` + `SECTION_TYPE_COLORS` in `lensConstants.js`.
- **Audio storage dir lifecycle (2026-06-21)**: `FilesystemAudioStorageAdapter` (`server/services/audioStorageService.js:61-118`) writes under `<uploadsRoot>/songs/{songId}.{ext}`. The constructor `mkdirSync`s `songs/` once at boot — that does NOT re-run if the dir is removed at runtime. The defensive `await fs.promises.mkdir(this.songsDir, { recursive: true })` inside `moveIntoStore` (line 89) was added after an operator-level rmdir of a root-owned `songs/` dir (left over from the prior root pm2 god daemon) caused the next re-download to fail with `ENOENT: copyfile /tmp/arra-dl-xxx/... -> /home/.../uploads/songs/...`. Regression test: `server/__tests__/unit/audioStorageService.test.js`. **If you ever `rmdir uploads/songs` on a running server, you must `pm2 restart arra-server`** — the running process's in-memory `songsDir` path won't re-materialize via the constructor until then, even though the defensive mkdir now makes the next download work without a restart.
